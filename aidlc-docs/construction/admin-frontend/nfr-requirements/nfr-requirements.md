# NFR Requirements — admin-frontend

## 성능
- **NFR-P-01**: SSE 이벤트 수신 후 UI 반영 16ms 이내 (단일 렌더 사이클)
- **NFR-P-02**: 대시보드 초기 로드 2초 이내 (API 응답 포함)
- **NFR-P-03**: 강조 애니메이션은 CSS transition 사용 (JS 루프 금지)

## 신뢰성
- **NFR-R-01**: SSE 재연결 — 끊김 감지 후 지수 백오프 (3s, 6s, 12s, 최대 5회)
- **NFR-R-02**: API 실패 시 토스트로 명확한 피드백, UI 롤백 없이 재시도 유도
- **NFR-R-03**: 모달 열린 상태에서 SSE 이벤트 수신 시 모달 데이터도 즉시 반영

## 보안
- **NFR-S-01**: JWT 토큰은 localStorage에 저장 (개발 환경 단순화)
- **NFR-S-02**: 모든 API 요청에 Authorization 헤더 자동 첨부
- **NFR-S-03**: 401 수신 즉시 토큰 삭제 + 로그인 화면 이동

## 사용성
- **NFR-U-01**: 신규 주문 강조 — 배경색 변경 + CSS 애니메이션 (3초)
- **NFR-U-02**: 버튼 최소 크기 44×44px (터치 대응)
- **NFR-U-03**: 로딩 중 버튼 비활성화 (중복 요청 방지)
- **NFR-U-04**: SSE 연결 상태 헤더에 상시 표시

## 유지보수성
- **NFR-M-01**: 빌드 도구 없음 (Vanilla JS, 즉시 실행)
- **NFR-M-02**: data-testid 속성으로 자동화 테스트 지원
- **NFR-M-03**: 모든 API 호출은 api.js 단일 모듈 경유
