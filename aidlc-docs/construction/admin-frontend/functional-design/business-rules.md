# 비즈니스 규칙 — admin-frontend

## 인증 규칙

### BR-AUTH-01: 토큰 유효성 선검사
- 페이지 로드 시 localStorage 토큰의 `exp` 클레임을 직접 파싱하여 만료 여부 확인
- 만료됐으면 API 호출 없이 즉시 로그인 화면으로

### BR-AUTH-02: API 401 자동 로그아웃
- 모든 API 요청에서 401 응답 수신 시 즉시 토큰 삭제 후 로그인 화면 이동
- SSE 연결도 함께 종료

### BR-AUTH-03: 로그인 폼 필수값 검증
- store_identifier, username, password 중 하나라도 비어있으면 API 호출 없이 에러 표시

## 대시보드 규칙

### BR-DASH-01: 신규 주문 강조 시간
- `new_order` SSE 이벤트 수신 시 해당 카드를 **3초간** 시각적으로 강조
- 3초 후 강조 자동 해제 (타이머 관리)
- 강조 중 새 주문 추가 시 타이머 초기화

### BR-DASH-02: 테이블 카드 정렬
- 항상 table_number 오름차순으로 표시
- SSE 이벤트로 데이터 변경 후에도 정렬 유지

### BR-DASH-03: 총 주문액 계산
- 클라이언트에서 직접 계산: `orders.reduce((sum, o) => sum + o.totalAmount, 0)`
- SSE 이벤트 수신 후 즉시 재계산하여 카드에 반영

### BR-DASH-04: 빈 테이블 표시
- 현재 세션 주문이 없는 테이블도 카드로 표시 (총액 0, 주문 없음 안내)

---

## 주문 관리 규칙

### BR-ORD-01: 상태 변경 버튼 조건부 표시
- PENDING → "준비 시작" 버튼만
- PREPARING → "완료 처리" 버튼만
- COMPLETED → 상태 변경 버튼 없음 (완료 표시만)

### BR-ORD-02: 삭제 이중 확인
- 주문 삭제는 반드시 확인 팝업 거친 후 실행

### BR-ORD-03: 모달 SSE 실시간 반영
- 주문 상세 모달이 열린 상태에서 SSE로 해당 주문 상태 변경/삭제 이벤트 수신 시 모달 즉시 업데이트
- 삭제 이벤트 수신 시 모달 자동 닫기

---

## 테이블 세션 규칙

### BR-SES-01: 이용 완료 이중 확인
- 테이블 번호를 포함한 확인 팝업: "{n}번 테이블을 이용 완료 처리하시겠습니까?"

### BR-SES-02: 이용 완료 후 UI 처리
- SSE `session_completed` 이벤트를 수신하면 해당 카드의 orders = [], totalAmount = 0
- API 응답만으로도 즉시 UI 반영 (SSE 이벤트 없어도 대응)

---

## SSE 규칙

### BR-SSE-01: 재연결 전략
- 연결 끊김 시 3초 후 첫 재연결, 이후 6초, 12초 (지수 백오프)
- 최대 5회 재시도 후 "연결이 끊겼습니다. 새로고침해 주세요" 안내

### BR-SSE-02: 연결 상태 표시
- 헤더 영역에 SSE 연결 상태 인디케이터 표시
- 연결됨: 초록 점 / 끊김: 빨간 점 + 재연결 중 문구

---

## UI 규칙

### BR-UI-01: 토스트 메시지
- 성공/실패 피드백은 토스트로 표시 (3초 후 자동 사라짐)
- 위치: 화면 우측 하단

### BR-UI-02: 로딩 상태
- API 요청 중 버튼 비활성화 (중복 요청 방지)
- 스피너 또는 "처리 중..." 텍스트 표시
