# Business Rules — customer-frontend

## 1. 인증 규칙

### BR-AUTH-001: 자동 로그인 조건
- localStorage에 `storeId`, `tableNumber`, `password` 세 값이 모두 존재할 때만 자동 로그인 시도
- 하나라도 누락 시 초기 설정 화면 표시

### BR-AUTH-002: 초기 설정 폼 유효성 검증
- 매장 식별자: 필수, 공백 불가
- 테이블 번호: 필수, 양의 정수
- 비밀번호: 필수, 공백 불가
- 모든 필드 입력 전 "설정 완료" 버튼 비활성화

### BR-AUTH-003: 인증 실패 처리
- 서버 401/403 응답: "인증 정보가 올바르지 않습니다. 다시 확인해 주세요." 표시
- 네트워크 오류: "서버에 연결할 수 없습니다. 잠시 후 다시 시도해 주세요." 표시
- 실패 시 입력값 유지 (비밀번호 필드만 초기화)

### BR-AUTH-004: 설정 재진입
- 메인 앱 진입 후 설정 화면으로 돌아갈 수 있는 버튼 제공
- 재진입 시 localStorage 인증 정보 삭제
- 장바구니 데이터는 유지 (인증과 분리)

---

## 2. 메뉴 조회 규칙

### BR-MENU-001: 메뉴 없음 처리
- API 응답이 빈 배열이거나 카테고리가 없는 경우: "등록된 메뉴가 없습니다" 메시지 표시

### BR-MENU-002: 이미지 로드 실패 처리 (Q11-C)
- 이미지 URL 없음 또는 `onerror` 발생 시: 회색 배경 박스만 표시 (텍스트 없음)
- CSS: `background-color: #e0e0e0`, 원래 이미지 `display: none`

### BR-MENU-003: 카테고리 탭 활성화
- 스크롤 시 현재 뷰포트에 보이는 카테고리 섹션에 해당하는 탭을 자동으로 활성화 (IntersectionObserver 활용)

### BR-MENU-004: 터치 영역 최소 크기
- 모든 버튼 및 터치 가능 영역: 최소 44x44px
- 태블릿 세로 모드 (Q12-A) 기준 레이아웃 최적화

## 3. 장바구니 규칙

### BR-CART-001: 수량 최솟값
- 수량 1에서 "-" 버튼 클릭 시 장바구니에서 해당 항목 제거 (0 표시 없음)

### BR-CART-002: 수량 중복 추가
- 이미 장바구니에 있는 메뉴 추가 시 수량 증가 (새 항목 추가 아님)

### BR-CART-003: 실시간 총액 계산
- 수량 변경 시 즉시 총액 재계산: `sum(item.price * item.quantity)`
- 천 단위 콤마 포맷: `toLocaleString('ko-KR')` + "원"

### BR-CART-004: 장바구니 탭 배지 (Q10-C)
- 장바구니에 아이템이 1개 이상일 때만 하단 탭 아이콘에 점(dot) 배지 표시
- 장바구니가 비어있으면 배지 숨김

### BR-CART-005: localStorage 동기화 시점
- CartManager 상태 변경 즉시 localStorage 동기화
- 페이지 로드 시 localStorage에서 복원 (stale 데이터 그대로 복원)

### BR-CART-006: 주문 성공 후 초기화
- 주문 API 성공 응답 수신 시 장바구니 즉시 초기화 (localStorage + 메모리)

### BR-CART-007: "장바구니 비우기" 버튼
- 클릭 시 전체 항목 삭제, localStorage 초기화, 총액 0으로 리셋

## 4. 주문 규칙

### BR-ORDER-001: 주문하기 버튼 활성화 조건
- 장바구니에 1개 이상의 아이템이 있을 때만 활성화
- 빈 장바구니: 버튼 비활성화 (disabled) + 시각적 그레이아웃

### BR-ORDER-002: 주문 확인 모달 (Q5-B)
- "주문하기" 클릭 시 모달 표시: 메뉴목록, 수량, 총액
- "확인": 주문 API 호출
- "취소": 모달 닫기, 장바구니 화면 유지

### BR-ORDER-003: 중복 주문 방지
- 주문 API 호출 중 "확인" 버튼 비활성화 (로딩 상태)
- API 응답 수신 후 버튼 재활성화

### BR-ORDER-004: 주문 실패 시 장바구니 유지
- 주문 API 실패 시 장바구니 데이터 변경 없음
- 모달 닫기 후 장바구니 화면에서 재시도 가능

### BR-ORDER-005: 주문 성공 피드백 (Q6-A)
- 성공 토스트 메시지: "주문이 완료되었습니다!" (3초 표시)
- 5초 후 자동으로 [메뉴 화면]으로 전환

### BR-ORDER-006: 오류 메시지 구분 (US-008)
- HTTP 0 / TypeError (fetch 실패): "네트워크 연결을 확인해 주세요."
- HTTP 4xx/5xx: "주문 처리 중 오류가 발생했습니다. 다시 시도해 주세요."

## 5. 주문 내역 규칙

### BR-HISTORY-001: 세션 기준 필터링
- API가 session_id 기준으로 필터링하여 반환 — 프론트엔드는 추가 필터 불필요
- 현재 localStorage의 sessionId를 API 요청에 포함하지 않음 (백엔드에서 토큰 기반 필터)

### BR-HISTORY-002: 주문 상태 표시 (Q7-B: Progress Bar)
- 대기중 (pending): Progress Bar 0% (빨간색 텍스트)
- 준비중 (preparing): Progress Bar 50% (노란색 텍스트)
- 완료 (completed): Progress Bar 100% (초록색 텍스트)
- 진행 바 CSS transition으로 부드럽게 업데이트

### BR-HISTORY-003: 주문 없음 처리
- 조회 결과 빈 배열: "아직 주문 내역이 없습니다." 안내 메시지

### BR-HISTORY-004: SSE 재연결 (Q8-A)
- 연결 끊김 시 3초 후 재연결 시도
- 최대 5회 재시도
- 5회 초과 시: "실시간 업데이트 연결이 끊겼습니다. 페이지를 새로고침해 주세요." 안내

### BR-HISTORY-005: SSE 중복 연결 방지
- SSE 연결은 앱 생명주기 동안 하나만 유지
- 이미 연결된 경우 재연결 시도 시 기존 연결 종료 후 재연결

## 6. 세션 초기화 규칙

### BR-SESSION-001: session_completed 이벤트 처리 (Q9-A)
- SSE `session_completed` 이벤트 수신 시:
  1. 주문 내역 UI 초기화 (빈 목록 표시)
  2. localStorage 장바구니 초기화
  3. 메모리 장바구니 초기화
  4. 메뉴 카드 수량 배지 초기화
  5. 하단 탭 장바구니 배지 숨김
  6. 화면 전환 없음 (현재 탭 유지)

### BR-SESSION-002: 새 세션 ID 처리
- `session_completed` 이후 다음 주문 시 백엔드가 자동으로 새 session_id 부여
- 프론트엔드는 새 세션 ID를 로그인 응답에서 획득 (재로그인 불필요)

## 7. UI/UX 규칙

### BR-UI-001: 태블릿 세로 모드 고정 (Q12-A)
- `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- CSS: `@media (orientation: landscape)` 로테이션 안내 오버레이 (선택적)

### BR-UI-002: 터치 피드백
- 모든 버튼/탭: CSS `active` 상태에서 시각적 피드백 (색상 변화 또는 scale 효과)

### BR-UI-003: 로딩 상태 표시
- API 호출 중: 버튼 비활성화 + 로딩 스피너 또는 텍스트 변경

### BR-UI-004: 에러 토스트
- 에러 발생 시 하단 토스트 메시지로 표시 (3초 후 자동 사라짐)
