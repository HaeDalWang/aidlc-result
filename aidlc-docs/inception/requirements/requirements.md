| 항목 | 내용 |
|---|---|

## 1. 기능 요구사항 (Functional Requirements)

### 1.1 고객용 기능 (Customer Features)

#### FR-C-01: 테이블 자동 로그인 및 세션 관리
- 관리자가 1회 수행하는 초기 설정: 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 최종 로그인 정보를 브라우저 localStorage에 저장
- 이후 접속 시 저장된 정보로 자동 로그인

#### FR-C-02: 메뉴 조회 및 탐색
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지 URL
- 카드 형태의 메뉴 레이아웃
- 터치 친화적 버튼 크기 (최소 44x44px)
- 카테고리 간 빠른 이동
- 메뉴 화면이 기본 화면으로 항상 표시

#### FR-C-03: 장바구니 관리
- 메뉴 추가/삭제 및 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- localStorage에 저장 (페이지 새로고침 시에도 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C-04: 주문 생성
- 주문 내역 최종 확인 화면
- 주문 확정 버튼
- 주문 성공 시: 주문 번호 표시(5초) → 장바구니 비우기 → 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보 포함: 매장 식별 정보, 테이블 식별 정보, 주문 메뉴 목록(메뉴명/수량/단가), 총 주문 금액, 세션 ID

#### FR-C-05: 주문 내역 조회
- 현재 테이블 세션의 주문만 표시 (이전 세션 제외)
- 주문 시간 순 정렬
- 주문별 정보: 주문 번호, 주문 시각, 주문 메뉴 및 수량, 주문 금액, 주문 상태
- **주문 상태 실시간 업데이트**: SSE(Server-Sent Events) 방식으로 실시간 반영
- 매장 이용 완료 처리된 주문은 표시 제외

---

### 1.2 관리자용 기능 (Admin Features)

#### FR-A-01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호로 로그인
- JWT 토큰 기반 인증 (16시간 세션 유지)
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱 저장

#### FR-A-02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 업데이트 (2초 이내 표시)
- 그리드/대시보드 레이아웃 (테이블별 카드 형태)
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 상세 정보: 테이블 번호, 주문 번호, 주문 시각, 주문 메뉴 및 수량, 총 금액
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경: 대기중 → 준비중 → 완료
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링 기능

#### FR-A-03: 테이블 관리
- **초기 설정**: 테이블 번호 + 테이블 비밀번호 설정, 자동 로그인 활성화
- **주문 삭제(직권 수정)**: 특정 주문 삭제, 확인 팝업, 테이블 총 주문액 재계산
- **테이블 세션 처리**:
  - 이용 완료(세션 종료) 처리: 확인 팝업 → 해당 세션의 주문 내역을 과거 이력으로 이동 → 현재 주문 목록 및 총 주문액 0으로 리셋
  - 세션 종료 후 새 고객이 이전 주문 내역 없이 시작 가능
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록(시간 역순), 날짜 필터링, 주문 번호/시각/메뉴 목록/총 금액/매장 이용 완료 시각 표시

#### FR-A-04: 메뉴 관리 (MVP 제외 — 읽기 전용)
- MVP에서는 메뉴 관리 UI 제공하지 않음
- 메뉴 데이터는 DB 시드 데이터 또는 직접 입력으로 관리
- 메뉴 이미지: 외부 URL 방식만 지원 (파일 업로드 없음)

## 2. 비기능 요구사항 (Non-Functional Requirements)

### 2.1 성능 (Performance)
- NFR-P-01: SSE를 통한 신규 주문 표시 시간 2초 이내
- NFR-P-02: API 응답 시간 1초 이내 (일반 조회)
- NFR-P-03: 동시 SSE 연결 최대 100개 지원

### 2.2 신뢰성 (Reliability)
- NFR-R-01: SSE 연결 끊김 시 클라이언트 자동 재연결
- NFR-R-02: 주문 처리 중 오류 시 사용자에게 명확한 에러 메시지 제공
- NFR-R-03: 장바구니 데이터 localStorage에 영속 저장

### 2.3 보안 (Security)
- NFR-S-01: 관리자 비밀번호 bcrypt 해싱 저장
- NFR-S-02: JWT 토큰 기반 관리자 인증 (16시간 만료)
- NFR-S-03: 테이블 인증 (매장 식별자 + 테이블 번호 + 테이블 비밀번호)
- NFR-S-04: 관리자 API는 JWT 토큰 검증 후에만 접근 허용

### 2.4 사용성 (Usability)
- NFR-U-01: 터치 친화적 UI (최소 버튼 크기 44x44px)
- NFR-U-02: 카드 기반 메뉴 레이아웃
- NFR-U-03: 신규 주문 시각적 강조 표시

### 2.5 유지보수성 (Maintainability)
- NFR-M-01: SQLite DB로 로컬 개발 환경 단순화
- NFR-M-02: DB 시드 데이터로 초기 매장 1개 자동 생성

## 3. 데이터 요구사항

### 핵심 엔티티
| 엔티티 | 설명 |
|---|---|
| Store | 매장 정보 (식별자, 이름, 관리자 계정) |
| Table | 테이블 정보 (번호, 비밀번호, 현재 세션 ID) |
| Menu | 메뉴 정보 (이름, 가격, 설명, 이미지 URL, 카테고리, 순서) |
| Category | 메뉴 카테고리 |
| Order | 주문 정보 (테이블, 메뉴 목록, 금액, 상태, 세션 ID) |
| OrderItem | 주문 항목 (메뉴, 수량, 단가) |
| OrderHistory | 과거 주문 이력 (세션 종료 시 이동) |

### 테이블 세션 개념
- 세션 ID: 고객이 테이블에 앉아 첫 주문부터 이용 완료 처리까지의 단위
- 세션 종료 시 주문 내역 → OrderHistory로 이동
- 새 고객 시작 시 새 세션 ID 생성

## 4. 시스템 경계 및 통합

### In-Scope (구현 대상)
- 고객용 웹 UI (Vanilla JavaScript)
- 관리자용 웹 UI (Vanilla JavaScript)
- FastAPI 백엔드 서버
- SQLite 데이터베이스
- SSE 실시간 통신

### Out-of-Scope (구현 제외 — constraints.md 참조)
- 결제 처리 (카드, 현금, 디지털 지갑, PG사 연동)
- 복잡한 사용자 인증 (OAuth, SNS, 2FA, OTP)
- 이미지 리사이징/최적화, CMS
- 푸시 알림, SMS, 이메일 알림
- 주방 기능 (주방 전달, 재고 관리)
- 데이터 분석, 매출 리포트
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어)

## 6. 전제 조건 및 제약사항

- 로컬 개발 환경 전용 (별도 배포 설정 없음)
- 단일 매장 기준으로 개발 (멀티 테넌시는 구조만 고려)
- SQLite 사용으로 인한 동시 쓰기 제한 인지 (중간 규모까지는 허용 범위)
- 메뉴 데이터는 개발자/관리자가 DB에 직접 입력
