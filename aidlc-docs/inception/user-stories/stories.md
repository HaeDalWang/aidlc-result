# 사용자 스토리 (User Stories)

**분류 방식**: Feature-Based (기능 단위)  
**Acceptance Criteria**: Standard (3~5개 조건 + 예외 상황)  
**INVEST 기준 준수**

---

## Feature 1: 테이블 초기 설정 및 자동 로그인

### US-001: 태블릿 초기 설정
**As a** 매장 관리자,  
**I want to** 태블릿에 매장 식별자, 테이블 번호, 테이블 비밀번호를 1회 입력하여 설정하고 싶다,  
**So that** 이후 고객이 별도 로그인 없이 즉시 주문 화면을 사용할 수 있다.

**Acceptance Criteria:**
- [ ] 매장 식별자, 테이블 번호, 테이블 비밀번호를 모두 입력하고 저장할 수 있다
- [ ] 저장된 인증 정보는 브라우저 localStorage에 유지된다
- [ ] 설정 완료 후 자동으로 메뉴 화면으로 이동한다
- [ ] 서버에서 인증 정보 검증 실패 시 명확한 에러 메시지를 표시한다
- [ ] 잘못된 매장 식별자 또는 테이블 비밀번호 입력 시 로그인이 거부된다

---

### US-002: 테이블 자동 로그인
**As a** 일반 고객,  
**I want to** 태블릿을 열면 자동으로 로그인되어 메뉴 화면이 바로 보이기를 원한다,  
**So that** 별도의 로그인 절차 없이 즉시 주문을 시작할 수 있다.

**Acceptance Criteria:**
- [ ] localStorage에 인증 정보가 저장된 경우 페이지 로드 시 자동으로 로그인된다
- [ ] 자동 로그인 성공 시 메뉴 화면이 기본 화면으로 표시된다
- [ ] 저장된 인증 정보가 없으면 초기 설정 화면으로 이동한다
- [ ] 서버 인증 실패 시 설정 화면으로 리다이렉트하고 에러 메시지를 표시한다

---

## Feature 2: 메뉴 조회

### US-003: 카테고리별 메뉴 탐색
**As a** 일반 고객,  
**I want to** 카테고리별로 분류된 메뉴 목록을 쉽게 탐색하고 싶다,  
**So that** 원하는 종류의 메뉴를 빠르게 찾을 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 화면이 기본(홈) 화면으로 항상 표시된다
- [ ] 상단 또는 측면에 카테고리 탭/버튼이 표시되고, 클릭 시 해당 카테고리 메뉴로 스크롤된다
- [ ] 각 메뉴는 카드 형태로 표시된다 (메뉴명, 가격, 이미지, 간단한 설명 포함)
- [ ] 모든 버튼/터치 영역은 최소 44x44px 이상이다
- [ ] 메뉴 또는 카테고리가 없을 경우 "등록된 메뉴가 없습니다" 안내 메시지를 표시한다

---

### US-004: 메뉴 상세 정보 확인
**As a** 일반 고객,  
**I want to** 메뉴 카드를 탭하여 상세 정보를 확인하고 싶다,  
**So that** 메뉴를 선택하기 전에 설명과 가격을 충분히 파악할 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 카드 탭 시 메뉴명, 가격, 상세 설명, 이미지가 표시된다
- [ ] 상세 화면에서 바로 장바구니에 추가할 수 있다
- [ ] 이미지 URL이 없거나 로드 실패 시 기본 플레이스홀더 이미지가 표시된다
- [ ] 뒤로가기 또는 닫기 버튼으로 메뉴 목록으로 돌아갈 수 있다

---

## Feature 3: 장바구니 관리

### US-005: 장바구니에 메뉴 추가 및 수량 변경
**As a** 일반 고객,  
**I want to** 원하는 메뉴를 장바구니에 담고 수량을 조절하고 싶다,  
**So that** 주문 확정 전에 원하는 메뉴 구성을 자유롭게 바꿀 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 추가 버튼(+) 클릭 시 장바구니에 해당 메뉴가 1개 추가된다
- [ ] 같은 메뉴를 여러 번 추가하면 수량이 증가한다
- [ ] 수량 감소(-) 버튼 클릭 시 수량이 줄어들며, 1에서 줄이면 장바구니에서 제거된다
- [ ] 장바구니 총 금액이 수량 변경 시 실시간으로 업데이트된다
- [ ] 장바구니 아이콘에 담긴 아이템 수 또는 총 금액이 표시된다

---

### US-006: 장바구니 데이터 영속성
**As a** 일반 고객,  
**I want to** 실수로 페이지를 새로고침해도 장바구니가 유지되기를 원한다,  
**So that** 장바구니에 담은 메뉴를 다시 선택하는 불편함이 없다.

**Acceptance Criteria:**
- [ ] 장바구니 데이터는 브라우저 localStorage에 저장된다
- [ ] 페이지 새로고침 후에도 장바구니 내용이 그대로 복원된다
- [ ] 주문 확정 성공 시 localStorage의 장바구니 데이터가 자동으로 초기화된다
- [ ] "장바구니 비우기" 버튼 클릭 시 전체 항목이 삭제된다

---

## Feature 4: 주문 생성

### US-007: 주문 확정 및 제출
**As a** 일반 고객,  
**I want to** 장바구니 내용을 최종 확인한 후 주문을 확정하고 싶다,  
**So that** 선택한 메뉴가 주방에 정확히 전달되었는지 확인할 수 있다.

**Acceptance Criteria:**
- [ ] 주문 확인 화면에서 메뉴 목록, 수량, 단가, 총 금액이 표시된다
- [ ] "주문하기" 버튼 클릭 시 주문이 서버로 전송된다
- [ ] 주문 성공 시 주문 번호가 표시되고 5초 후 메뉴 화면으로 자동 이동한다
- [ ] 주문 성공 후 장바구니가 자동으로 비워진다
- [ ] 장바구니가 비어있을 경우 주문하기 버튼이 비활성화된다

---

### US-008: 주문 실패 처리
**As a** 일반 고객,  
**I want to** 주문이 실패했을 때 명확한 안내를 받고 싶다,  
**So that** 장바구니 내용을 잃지 않고 다시 주문을 시도할 수 있다.

**Acceptance Criteria:**
- [ ] 서버 오류로 주문 실패 시 사용자에게 에러 메시지를 표시한다
- [ ] 주문 실패 시 장바구니 데이터는 그대로 유지된다
- [ ] 다시 시도할 수 있도록 "재시도" 버튼 또는 안내 문구를 제공한다
- [ ] 네트워크 오류와 서버 오류를 구분하여 안내한다

---

## Feature 5: 주문 내역 조회 (고객)

### US-009: 현재 세션 주문 내역 확인
**As a** 일반 고객,  
**I want to** 내가 이번에 주문한 내역과 현재 상태를 확인하고 싶다,  
**So that** 음식이 언제 나오는지, 주문이 제대로 접수됐는지 파악할 수 있다.

**Acceptance Criteria:**
- [ ] 현재 테이블 세션에서 생성된 주문만 표시된다 (이전 세션 주문 제외)
- [ ] 주문별로 주문 번호, 주문 시각, 메뉴 목록, 총 금액, 주문 상태가 표시된다
- [ ] 주문 상태(대기중/준비중/완료)가 SSE를 통해 실시간으로 업데이트된다
- [ ] 주문이 없을 경우 "아직 주문 내역이 없습니다" 메시지가 표시된다
- [ ] SSE 연결 끊김 시 자동으로 재연결을 시도한다

---

## Feature 6: 테이블 세션 — 고객 관점

### US-010: 새 고객의 깨끗한 시작
**As a** 일반 고객,  
**I want to** 테이블에 앉았을 때 이전 손님의 주문 내역이 보이지 않기를 원한다,  
**So that** 내 주문 내역만 정확하게 관리할 수 있다.

**Acceptance Criteria:**
- [ ] 관리자가 테이블 이용 완료 처리 후 새 세션이 시작되면 주문 내역 화면이 비어있다
- [ ] 새 세션의 주문만 주문 내역 화면에 표시된다
- [ ] 이전 세션의 주문은 고객 화면에서 접근할 수 없다
- [ ] 주문 내역 조회 시 현재 세션 ID 기준으로 필터링된다

---

## Feature 7: 매장 관리자 인증

### US-011: 관리자 로그인
**As a** 매장 관리자,  
**I want to** 매장 식별자, 사용자명, 비밀번호로 관리자 화면에 로그인하고 싶다,  
**So that** 승인된 관리자만 주문 관리 기능에 접근할 수 있다.

**Acceptance Criteria:**
- [ ] 매장 식별자, 사용자명, 비밀번호 입력 후 로그인이 성공하면 관리자 대시보드로 이동한다
- [ ] 로그인 성공 시 JWT 토큰이 발급되어 브라우저에 저장된다
- [ ] 잘못된 자격 증명 입력 시 "인증 정보가 올바르지 않습니다" 에러 메시지를 표시한다
- [ ] 비밀번호는 화면에 마스킹(****) 처리된다
- [ ] 로그인 실패 반복 시 접근이 제한된다 (rate limiting)

---

### US-012: 관리자 세션 유지 및 자동 로그아웃
**As a** 매장 관리자,  
**I want to** 브라우저를 새로고침해도 로그인 상태가 유지되고 16시간 후 자동 로그아웃되기를 원한다,  
**So that** 영업 시간 동안 재로그인 없이 편리하게 사용하고, 영업 종료 후에는 보안이 유지된다.

**Acceptance Criteria:**
- [ ] 브라우저 새로고침 후에도 JWT 토큰이 유효하면 로그인 상태가 유지된다
- [ ] JWT 토큰 발급 후 16시간이 지나면 자동으로 로그아웃되고 로그인 화면으로 이동한다
- [ ] 토큰 만료 시 사용자에게 "세션이 만료되었습니다" 안내 메시지를 표시한다
- [ ] 로그아웃 버튼 클릭 시 즉시 토큰이 삭제되고 로그인 화면으로 이동한다

---

## Feature 8: 실시간 주문 모니터링

### US-013: 신규 주문 실시간 확인
**As a** 매장 관리자,  
**I want to** 새로운 주문이 들어오면 대시보드에서 즉시 확인하고 싶다,  
**So that** 주문 누락 없이 신속하게 대응할 수 있다.

**Acceptance Criteria:**
- [ ] 신규 주문 발생 시 2초 이내에 관리자 대시보드에 표시된다
- [ ] 새로 들어온 주문은 색상 변경 또는 애니메이션으로 시각적으로 강조된다
- [ ] 각 테이블 카드에 총 주문액과 최신 주문 내역 미리보기가 표시된다
- [ ] SSE 연결 끊김 시 자동 재연결을 시도하고 상태를 화면에 표시한다
- [ ] 테이블별 필터링 기능으로 특정 테이블의 주문만 볼 수 있다

---

### US-014: 주문 상세 조회 및 상태 변경
**As a** 매장 관리자,  
**I want to** 주문 카드를 클릭하여 전체 메뉴 목록을 확인하고 상태를 변경하고 싶다,  
**So that** 주문 처리 진행 상황을 정확하게 추적하고 관리할 수 있다.

**Acceptance Criteria:**
- [ ] 주문 카드 클릭 시 전체 메뉴 목록, 수량, 총 금액이 상세 팝업으로 표시된다
- [ ] 주문 상태를 "대기중 → 준비중 → 완료" 순으로 변경할 수 있다
- [ ] 상태 변경 시 고객 화면의 주문 내역에도 즉시 반영된다 (SSE)
- [ ] 상태 변경 성공/실패 피드백이 표시된다

---

## Feature 9: 테이블 관리 (관리자)

### US-015: 주문 삭제 (직권 수정)
**As a** 매장 관리자,  
**I want to** 잘못된 주문을 삭제하고 싶다,  
**So that** 실수로 들어온 주문이 총 주문액에 반영되지 않도록 수정할 수 있다.

**Acceptance Criteria:**
- [ ] 각 주문에 삭제 버튼이 표시된다
- [ ] 삭제 버튼 클릭 시 "정말 삭제하시겠습니까?" 확인 팝업이 표시된다
- [ ] 확인 후 주문이 즉시 삭제되고 테이블 총 주문액이 재계산된다
- [ ] 삭제 성공/실패 피드백이 표시된다
- [ ] 삭제된 주문은 고객 화면의 주문 내역에서도 제거된다

---

## Feature 10: 테이블 세션 관리 — 관리자 관점

### US-016: 테이블 이용 완료 처리 (관리자)
**As a** 매장 관리자,  
**I want to** 고객 계산 후 테이블 이용 완료 처리를 하고 싶다,  
**So that** 현재 테이블의 주문 내역이 초기화되고 다음 고객을 위한 깨끗한 상태가 된다.

**Acceptance Criteria:**
- [ ] "이용 완료" 버튼 클릭 시 "테이블을 초기화하시겠습니까?" 확인 팝업이 표시된다
- [ ] 확인 시 해당 세션의 주문 내역이 과거 이력(OrderHistory)으로 이동한다
- [ ] 테이블의 현재 주문 목록과 총 주문액이 0으로 리셋된다
- [ ] 새 고객이 주문을 시작하면 새로운 세션 ID로 추적된다
- [ ] 처리 성공/실패 피드백이 표시된다

---

### US-017: 테이블 이용 완료 처리 — 고객 관점에서의 영향
**As a** 일반 고객 (새로 착석),  
**I want to** 테이블에 앉으면 이전 손님의 주문 내역이 전혀 보이지 않기를 원한다,  
**So that** 내 주문만 깔끔하게 관리할 수 있다.

**Acceptance Criteria:**
- [ ] 관리자 이용 완료 처리 후 첫 주문 시 새 세션 ID가 생성된다
- [ ] 새 세션의 고객 주문 내역 화면은 빈 상태로 시작된다
- [ ] 이전 세션 주문은 고객 화면에서 접근 불가하다
- [ ] 자동 로그인 후에도 세션 기준으로 주문 내역이 필터링된다

---

### US-018: 과거 주문 내역 조회 (관리자)
**As a** 매장 관리자,  
**I want to** 특정 테이블의 과거 주문 이력을 조회하고 싶다,  
**So that** 매출 확인이나 분쟁 발생 시 기록을 참조할 수 있다.

**Acceptance Criteria:**
- [ ] "과거 내역" 버튼 클릭 시 해당 테이블의 이전 세션 주문 목록이 시간 역순으로 표시된다
